# Найти идею. Введение в ТРИЗ - Г. Альтшуллер - supporting fragment toolkit

Статус: quality-first fragment-only toolkit для staged multi-book TRIZ workflow. Используется как supporting source layer `ALT-FRAG`, но не как самостоятельная полная книга.

Важно: файл содержит только ознакомительный фрагмент. Поэтому этот toolkit намеренно ограничен: он извлекает вводный ethos TRIZ, но не подменяет полные sources по ARIZ, standards, matrix, S-field или full methodology.

## 2. Как пользоваться toolkit

Открывай этот fragment toolkit, когда нужно:
- быстро объяснить ethos TRIZ: к идеалу без компромиссов;
- показать отличие TRIZ от метода проб и ошибок;
- ввести психологическую инерцию and need for systematic invention;
- дать легкий вход в тему перед полными sources.

Не используй этот файл:
- как единственный источник для ARIZ-85В;
- как источник полной матрицы, 40 приемов or standards;
- как доказательство coverage по всей книге;
- как substitute for full `ALT-RU-ALG`, `ALT-EN-ALG`, `ALT-EXACT`, `GADD`, `ORLOV`, `HLYN`.

## 4. Лайфхаки, приемы и инструменты к внедрению

### 4.1 Начинать с "идеала без компромисса"

Что внедрить:
- В начале каждой TRIZ-сессии строку `идеально, если результат достигается без затрат, вреда и добавления системы`.

Когда применять:
- Когда команда слишком рано соглашается на "чуть лучше, но с новым минусом".

Первый шаг:
- Спросить: "какой вред должен исчезнуть сам или начать приносить пользу?"

Источник / где искать в книге:
- Source copy: `Предисловие. К идеалу - без компромиссов!`, search `Идеальное решение`, `ИКР`.

### 4.2 Использовать фрагмент как motivation layer, not method layer

Что внедрить:
- Метку `supporting only` в coverage notes.

Когда применять:
- При combined synthesis, чтобы не завысить вес неполного источника.

Первый шаг:
- Для каждой идеи из фрагмента найти подтверждение в полном source before using as key conclusion.

Источник / где искать в книге:
- Source metadata: `ознакомительный фрагмент`, `Конец ознакомительного фрагмента`.

### 4.3 Останавливать метод проб и ошибок вопросом о закономерности

Что внедрить:
- После каждой новой пробы спрашивать: "какую закономерность или противоречие мы проверяем?"

Когда применять:
- Когда команда хочет продолжать перебор без learning.

Первый шаг:
- Записать текущий список проб и отметить, какие assumptions они проверяли.

Источник / где искать в книге:
- Source copy: `1. Страшнее ураганов`, `2. Бунт на коленях`.

### 4.4 Делать психологическую инерцию явной

Что внедрить:
- Список привычных терминов and forbidden options.

Когда применять:
- Когда "невозможно" основано на текущей форме объекта.

Первый шаг:
- Переписать задачу без названия устройства and material.

Источник / где искать в книге:
- Source copy introductory text, search `психологической инерции`, `старых представлений`.

### 4.5 Подключать полный source before method detail

Что внедрить:
- Rule: if action requires ARIZ/matrix/standards, source must switch from `ALT-FRAG` to full source.

Когда применять:
- В любом combined toolkit section, где fragment idea becomes method instruction.

Первый шаг:
- Add traceability to `ALT-RU-ALG`, `ALT-EN-ALG`, `ALT-EXACT`, `GADD`, `ORLOV` or `HLYN`.

Источник / где искать в книге:
- This fragment toolkit `Scope and limits`; full source manifests.

## 5. Карта фрагмента по смысловым блокам

### Блок A. Вводная рамка TRIZ

Core idea:
- TRIZ presented as a theory that makes invention manageable through laws and disciplined thinking.

Practical use:
- Use as opening explanation for why Books combined toolkit is not "creative tips".

Limit:
- Does not provide full methods.

### Блок B. К идеалу без компромиссов

Core idea:
- The strongest target is not a trade-off, but an ideal result where harm disappears or becomes useful.

Practical use:
- Use as mindset before contradiction work.

Limit:
- For operational IFR use `ALT-INVENTOR`, `GADD`, `HLYN`.

### Блок C. Against trial-and-error

Core idea:
- Growth of technical complexity requires a different technology of improvement than pure search.

Practical use:
- Use to justify moving from brainstorming to models.

Limit:
- For detailed levels/matrix use `ALT-EN-ALG` and `ALT-RU-ALG`.

### Блок D. Psychological inertia

Core idea:
- Old representations make inventive possibilities invisible.

Practical use:
- Use to frame anti-inertia exercises.

Limit:
- For actual operators use `ALT-INVENTOR` STC and little-men; `ORLOV` fantasy/reinvention.

## 7. Принципы применения

1. Use `ALT-FRAG` for framing, not for method authority.
2. Never use fragment-only source as sole evidence for a key combined rule.
3. Convert ideal language into concrete contradiction/IFR through full sources.
4. Convert anti-trial-and-error language into process rules through full sources.
5. Mark fragment contribution explicitly in coverage map.

## 8. Patterns / techniques

### 8.1 Fragment-safe use

1. Extract mindset idea.
2. Label it `supporting`.
3. Find full-source confirmation.
4. Use full source for method detail.
5. Keep fragment as additional context.

### 8.2 Ideal kickoff

1. Ask for desired result.
2. Remove current implementation.
3. Remove compromise.
4. Ask what harm disappears.
5. Continue in full source route.

### 8.3 Trial-and-error reset

1. List attempts.
2. List assumptions.
3. Identify repeated pattern.
4. Move to contradiction model.
