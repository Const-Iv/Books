## 2. Как пользоваться toolkit

Открывай этот toolkit, когда нужно:
- заменить тяжелый полный ARIZ более управляемой A-navigation;
- диагностировать problem situation before choosing tool;
- найти operational zone and resources;
- работать со спорным matrix mapping через CICO;
- выбрать A-navigator rather than a random principle;
- верифицировать решение и вторичные задачи;
- обучать TRIZ через reinvention;
- внедрять TRIZ в профессию, а не проводить разовый workshop.

Практический маршрут:
1. Диагностируй problem situation.
2. Найди operational zone.
3. Собери resources.
4. Выбери Meta-ARIZ route or A-navigator.
5. Если matrix mapping спорный, используй CICO.
6. Сгенерируй concepts.
7. Сделай verification and secondary problem pass.
8. Сохрани case as reinvention/training artifact.

## 4. Лайфхаки, приемы и инструменты к внедрению

### 4.1 Делать diagnosis gate до выбора инструмента

Что внедрить:
- Gate: `тип проблемной ситуации`, `что известно`, `что неизвестно`, `какой уровень модели нужен`.

Когда применять:
- Когда команда сразу тянется к matrix, standards or ARIZ.

Первый шаг:
- Записать problem situation как symptom cluster, then ask which symptom is central.

Источник / где искать в книге:
- Source copy: `16. Диагностика проблемы`, `16.2. Алгоритм диагностики проблемной ситуации`.

### 4.2 Фиксировать operational zone как эпицентр

Что внедрить:
- Поле `ОЗ`: минимальное место и условия, где конфликт появляется.

Когда применять:
- Когда анализ расползается на всю систему.

Первый шаг:
- Сузить проблему до контакта, поля, границы, интерфейса, момента или режима.

Источник / где искать в книге:
- Source copy: `8. Оперативная зона`, `8.1. Эпицентр проблемы`.

### 4.3 Делать resource scan именно в operational zone

Что внедрить:
- Список ресурсов: вещества, поля, форма, время, вредные факторы, соседние элементы, информация, human/organizational resources.

Когда применять:
- Перед добавлением нового компонента.

Первый шаг:
- Для каждого ресурса указать: `уже есть`, `можно высвободить`, `можно преобразовать`.

Источник / где искать в книге:
- Source copy: `8.2. Ресурсы`.

### 4.4 Использовать Meta-ARIZ как компактный маршрут

Что внедрить:
- Route: `diagnosis -> OZ/resources -> contradiction -> FIM/model -> navigator -> concept -> verification`.

Когда применять:
- Когда задача сложная, но полный ARIZ слишком длинный для команды.

Первый шаг:
- Выбрать, с какой части маршрута начинать: symptom, contradiction, resource or model.

Источник / где искать в книге:
- Source copy: `7.2. Мета-Алгоритм Изобретения`, search `Мета-АРИЗ`.

### 4.5 Использовать CICO при спорном mapping

Что внедрить:
- Cluster input / cluster output: несколько альтернативных параметрических входов и рейтинг повторяющихся principles/navigators.

Когда применять:
- Когда один conflict можно честно описать несколькими parameter pairs.

Первый шаг:
- Сделать 3-5 pairs; собрать recommended directions; выбрать повторяющиеся.

Источник / где искать в книге:
- Source copy: `11.4. Интеграция альтернативных противоречий - метод CICO`.

### 4.6 Переводить противоречие в functional ideal model

Что внедрить:
- FIM-карточку: `X-resource does useful function in OZ without negative effect`.

Когда применять:
- Когда нужна не просто идея, а model of how contradiction is eliminated.

Первый шаг:
- Записать X-resource and what it must do without increasing system complexity.

Источник / где искать в книге:
- Source copy: search `ФИМ`, `Функциональное идеальное моделирование`.

### 4.7 Верифицировать concept отдельным алгоритмом

Что внедрить:
- Verification table: `concept`, `contradiction solved`, `resource`, `new harm`, `secondary problem`, `test`.

Когда применять:
- После первого strong concept.

Первый шаг:
- Проверить, исчез conflict or it only moved to another subsystem.

Источник / где искать в книге:
- Source copy: `17. Верификация решения`, `17.3. Алгоритм верификации решения`.

### 4.8 Делать reinvention как метод обучения

Что внедрить:
- Exercise: hide final known solution, reconstruct problem route.

Когда применять:
- При обучении TRIZ, анализе патентов/конкурентов, ретроспективе сильных решений.

Первый шаг:
- Взять known solution, удалить ответ, записать исходные symptoms and contradiction.

Источник / где искать в книге:
- Source copy: `19.4. Примеры реинвентинга`, search `реинвентинг`.

### 4.9 Проверять "десять типичных ошибок" перед workshop

Что внедрить:
- Risk checklist: wrong diagnosis, no OZ, resource blindness, mechanical matrix, no verification, no adaptation to profession.

Когда применять:
- Перед первой TRIZ-сессией в команде.

Первый шаг:
- Выбрать 3 наиболее вероятные ошибки конкретной команды and write guardrails.

Источник / где искать в книге:
- Source copy: `19.3. Десять типичных ошибок`.

### 4.10 Адаптировать TRIZ к профессии

Что внедрить:
- Mapping: `role tasks -> typical contradictions -> preferred navigators -> case library`.

Когда применять:
- Когда TRIZ нужно встроить в регулярную работу, а не провести разовое обучение.

Первый шаг:
- Выписать 5 повторяющихся задач профессии and map each to TRIZ route.

Источник / где искать в книге:

## 5. Карта книги по смысловым блокам

### Блок A. A-studio and A-navigation

Core idea:
- Thinking can be navigated with structured categories and navigators, not only with a fixed algorithm.

Key concepts:
- A-navigation;
- A-navigators;
- algorithmic navigation of thinking.

Практический вывод:
- Choose navigator based on problem state, not habit.

### Блок B. Meta-ARIZ

Core idea:
- Full ARIZ can be abstracted into a meta-route that keeps structure while becoming more usable.

Key concepts:
- diagnosis;
- operational zone;
- resource;
- contradiction;
- functional ideal modeling;
- navigator choice.

Практический вывод:
- Meta-ARIZ is the bridge between "too much ARIZ" and shallow matrix.

### Блок C. Operational zone and resources

Core idea:
- Resources matter most where conflict actually happens.

Key concepts:
- epicenter of problem;
- operative resources;
- X-resource.

Практический вывод:
- Never scan resources globally before defining OZ.

### Блок D. Contradictions and CICO

Core idea:
- Matrix mapping can be ambiguous; CICO handles ambiguity by using several honest entries and clustering outputs.

Key concepts:
- inverse contradictions;
- alternative contradictions;
- cluster input/output;
- specialized navigators.

Практический вывод:
- If parameter choice is political or fuzzy, use CICO.

### Блок E. Verification and solution development

Core idea:
- A concept is not complete until it is checked for solved contradiction, resources and secondary problems.

Key concepts:
- effectiveness of solution;
- development of solution;
- verification algorithm.

Практический вывод:
- Verification is not QA after implementation; it is part of inventive thinking.

### Блок F. Art and profession of invention

Core idea:
- TRIZ becomes durable when it is adapted to professional tasks and practiced through reinvention.

Key concepts:
- fantasy/practical imagination;
- little figures;
- typical mistakes;
- reinvention;
- professional adaptation.

Практический вывод:
- Build a role-specific case library.

## 7. Принципы применения

1. Diagnosis before navigator.
2. Operational zone before resources.
3. Resources before new components.
4. CICO before arguing about one parameter pair.
5. FIM before concept selection.
6. Verification before calling a concept strong.
7. Reinvention before abstract lectures.
8. Professional adaptation before rollout.

## 8. Patterns / techniques

### 8.1 Meta-ARIZ mini-route

1. Describe problem situation.
2. Diagnose type.
3. Locate operational zone.
4. Scan resources in OZ.
5. State contradiction.
6. Build FIM.
7. Choose navigator.
8. Generate concept.
9. Verify and develop.

### 8.2 CICO route

1. Write one conflict.
2. Create 3-5 alternative input pairs.
3. Run each through matrix/navigator logic.
4. Cluster outputs.
5. Keep repeated directions.
6. Translate into concepts.
7. Verify against original contradiction.

### 8.3 Verification table

| Field | Question |
|---|---|
| Contradiction solved | What conflict disappeared? |
| Resource | What existing resource does work? |
| New harm | What became worse? |
| Secondary problem | What must be solved next? |
| Test | What observation proves concept? |

### 8.4 Reinvention route

1. Select known strong solution.
2. Hide final answer.
3. Recover initial problem.
4. Build contradiction.
5. Choose navigator.
6. Generate candidate.
7. Compare with actual solution.
8. Extract principle.

## 9. Anti-patterns

| Anti-pattern | Симптом | Почему плохо | Коррекция |
|---|---|---|---|
| Meta-ARIZ as checklist | steps filled mechanically | no thinking change | require output from each step |
| Global resource scan | huge list, no leverage | resources not in conflict zone | OZ first |
| One mapping fight | debate over parameter names | low signal | CICO |
| FIM skipped | concepts are gadgets | no ideality model | X-resource formulation |
| Verification skipped | strong-sounding idea fails later | hidden new harms | verification table |
| TRIZ not adapted to role | after training no one uses it | no work integration | professional task mapping |
