## 2. Как пользоваться toolkit

Открывай этот toolkit, когда нужно:
- обучить команду TRIZ через задачи, а не лекцию;
- показать переход от "невозможно" к technical contradiction;
- объяснить technical system and S-field alphabet простым языком;
- использовать ideal final result, STC and little-men modeling against inertia;
- построить тренировочную картотеку задач.

Практический маршрут:
1. Возьми задачу, которая кажется невозможной.
2. Найди technical contradiction.
3. Покажи, что задача живет внутри technical system.
4. Если функция слабая, построй S-field alphabet.
5. Для инерции включи STC.
6. Для микроуровня включи little-men model.
7. Закрепи через task card and answer comparison.

## 4. Лайфхаки, приемы и инструменты к внедрению

### 4.1 Начинать TRIZ workshop с "невозможной" задачи

Что внедрить:
- Opening exercise: короткая задача, где обычный подход застревает, а contradiction делает путь видимым.

Когда применять:
- Для новичков, скептиков и команд, которые считают TRIZ набором абстрактных терминов.

Первый шаг:
- Запретить решения на 5 минут и попросить участников назвать, что должно улучшиться и что при этом ухудшается.

Источник / где искать в книге:
- Source copy: OCR page 006, `Part 1: The Beginning of the Theory`, `Chapter 1: It's Impossible`.

### 4.2 Делать contradiction card из каждого примера

Что внедрить:
- Формат: `want X`, `but Y worsens`, `where conflict happens`, `what must be separated`.

Когда применять:
- В обучении и в реальной problem-solving сессии.

Первый шаг:
- После описания задачи не спрашивать "какие идеи?", а спрашивать "какой trade-off?"

Источник / где искать в книге:
- Source copy: OCR page 006, `Chapter 3: Technical Contradictions`.

### 4.3 Показывать technical system до детали

Что внедрить:
- Мини-карта: subsystem / system / supersystem, полезная функция, вред, среда.

Когда применять:
- Когда команда спорит о компоненте и не видит надсистемный ресурс.

Первый шаг:
- Записать, ради какой функции существует система.

Источник / где искать в книге:
- Source copy: OCR page 006, `Part 2: The Era of Technical Systems`.

### 4.4 Использовать S-field alphabet как первый model layer

Что внедрить:
- Простую азбуку interaction: object, tool, field, useful/harmful/weak action.

Когда применять:
- Когда function analysis GADD кажется слишком тяжелым для начального обучения.

Первый шаг:
- Нарисовать один action link and mark what is missing.

Источник / где искать в книге:
- Source copy: OCR page 006, `Chapter 9`, `Chapter 10`.

### 4.5 Формулировать IFR как учебную дисциплину

Что внедрить:
- В каждой задаче строку: `идеально, если нужный результат получается сам, без добавления системы`.

Когда применять:
- Когда participants предлагают компромисс или новый механизм.

Первый шаг:
- Спросить, какой элемент должен исчезнуть, сохранив функцию.

Источник / где искать в книге:
- Source copy: OCR page 007, `Chapter 18: Head for the Ideal Final Result`.

### 4.6 Прогонять инерцию через STC

Что внедрить:
- Size-Time-Cost extremes: object tiny/huge, process instant/slow, cost zero/infinite.

Когда применять:
- Когда все варианты похожи на существующую конструкцию.

Первый шаг:
- Взять один параметр STC и довести его до абсурда, затем вернуть полезный принцип в реальность.

Источник / где искать в книге:
- Source copy: OCR page 007, `Chapter 23: Strange Mirrors of the Operator STC`.

### 4.7 Использовать little-men model для "невидимой" физики

Что внедрить:
- Микро-сценарий, где множество маленьких исполнителей делает нужное действие.

Когда применять:
- Контакт, сцепление, поток, порошки, поля, жидкости, распределенная нагрузка.

Первый шаг:
- Спросить: "что каждый маленький исполнитель должен сделать, чтобы macro function появилась?"

Источник / где искать в книге:
- Source copy: OCR page 007, `Chapter 24`.

### 4.8 Вести task card library

Что внедрить:
- Карточку: `condition`, `contradiction`, `tool`, `concept`, `answer comparison`, `lesson`.

Когда применять:
- Для обучения, ретроспективы и развития команды.

Первый шаг:
- После каждой задачи сохранять не только ответ, но и route.

Источник / где искать в книге:
- Source copy: `Part 6: The Amazing World of Tasks`, appendices.

### 4.9 Разделять training mode and battle mode

Что внедрить:
- Два режима: учебный полный разбор и рабочий короткий route.

Когда применять:
- Когда команда впервые применяет TRIZ на реальной задаче.

Первый шаг:
- Для боевой задачи брать только contradiction -> IFR -> tool; полный task drill оставлять для обучения.

Источник / где искать в книге:
- Source copy: practical chapters and task appendices.

### 4.10 Переводить каждый пример в transferable principle

Что внедрить:
- После примера записывать: "какой общий прием или model здесь был применен?"

Когда применять:
- Чтобы учебные задачи не остались anecdotes.

Первый шаг:
- Убрать отраслевые детали и назвать transformation: separation, field completion, prior action, function transfer, micro-level.

Источник / где искать в книге:
- Source copy: task chapters and appendices.

## 5. Карта книги по смысловым блокам

### Part 1. Beginning of theory

Core idea:
- TRIZ начинается с демонстрации, что "невозможная" задача имеет hidden contradiction.

Key concepts:
- impossible problem;
- simple examples;
- technical contradictions;
- thinking for yourself.

Практический вывод:
- Начинать обучение нужно с изменения способа видеть задачу.

### Part 2. Era of technical systems

Core idea:
- Изобретение происходит не в изолированной детали, а в technical system with functions and interactions.

Key concepts:
- system perspective;
- periods of system development;
- M-field and S-field alphabet.

Практический вывод:
- Простая S-field карта быстрее вводит новичков в function thinking.

### Part 3. Science of inventing

Core idea:
- Strong invention requires physics and principle of action, not just cleverness.

Key concepts:
- physical effects;
- ideal final result;
- order in brain's attic.

Практический вывод:
- После contradiction надо искать не "идею", а physical route.

### Part 4. Art of inventing

Core idea:
- Inertia breaks through mental operators that change scale and representation.

Key concepts:
- STC;
- little-men modeling;
- ideal machine.

Практический вывод:
- STC and little men are not games; they are ways to reveal hidden resources and micro-actions.

### Part 5. Algorithm of talent

Core idea:
- Talent can be supported by problem modeling and algorithmic discipline.

Key concepts:
- problem model;
- substance/no-substance trick;
- stubborn problems.

Практический вывод:
- Store not just answers, but solution paths.

### Part 6. World of tasks

Core idea:
- Skill grows through tasks, answer comparison and extraction of transferable principles.

Key concepts:
- task library;
- answers;
- methods/effects/tricks.

Практический вывод:
- A TRIZ team needs a case library, not only a glossary.

## 7. Принципы применения

1. Teach TRIZ through tasks before terminology.
2. Every example must produce a contradiction card.
3. A system view comes before component redesign.
4. S-field alphabet is enough for first modeling pass.
5. IFR must challenge compromise.
6. STC extremes should return to real resources.
7. Little-men model must translate back into material/field/structure.
8. Save solution path, not just answer.

## 8. Patterns / techniques

### 8.1 Training task route

1. Present condition.
2. Ask for contradiction.
3. Ask for IFR.
4. Choose one tool.
5. Generate concept.
6. Compare with answer.
7. Extract transferable principle.

### 8.2 STC route

| Dimension | Extreme questions |
|---|---|
| Size | What if object is microscopic? Huge? Divided into many? |
| Time | What if action happens before conflict? Instantly? Very slowly? |
| Cost | What if cost is zero? Infinite? What if resource is free because it already exists? |

### 8.3 Little-men route

1. Replace object with crowd of agents.
2. Give agents local rules.
3. Make global function emerge.
4. Translate rules to structure/material/field.

### 8.4 Example-to-principle route

1. Remove domain nouns.
2. Keep function and contradiction.
3. Identify transformation.
4. Store as pattern.

## 12. Сценарии применения

### 12.1 Вводный TRIZ workshop

Use:
- impossible opener;
- contradiction card;
- IFR;
- one STC exercise;
- task card library.

Output:
- participants can name contradiction and one route, not just repeat definitions.

### 12.2 Разбор застрявшей product/engineering задачи

Use:
- technical system map;
- S-field alphabet;
- IFR;
- STC or little-men model.

Output:
- at least 3 concepts tied to hidden resource or separated contradiction.

### 12.3 Создание учебной базы

Use:
- task route;
- answer comparison;
- example-to-principle extraction.

Output:
- library of reusable cases for later team practice.
